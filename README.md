# Rooms

Rooms — рабочий прототип адаптивного сайта для поиска и бронирования приватных помещений: лаунджей, детских комнат, караоке-комнат, банкетных залов, переговорных и VIP-комнат.

Стартовый город MVP: Воронеж.

## Что сейчас есть

- Главная страница с подбором помещения по дате, времени, длительности и количеству гостей.
- Каталог демо-помещений с фильтрами, избранным, режимами «Список / Карта» и сохранением города поиска.
- Определение ближайшего поддерживаемого города через геопозицию браузера.
- Логика 18+: кальян и алкоголь скрываются, если среди гостей есть несовершеннолетние.
- Альтернативные окна, если выбранное время занято.
- Карточка помещения с фотогалереей, картой, отзывами, допуслугами и похожими вариантами.
- Заявка с несколькими помещениями, расчётом полной стоимости, предоплаты 30% и остатка на месте.
- Личный кабинет клиента со статусами, чатом, оплатой, чеками, отменами и отзывами после посещения.
- Кабинет партнёра с календарём, ручными бронями, заявками, чатами, помещениями, фото, тарифом и выплатами.
- Админка Rooms с модерацией, очередями, статистикой, оплатами, рейтингом, уведомлениями и историей действий.
- Отдельный кабинет бухгалтера для сверки и запуска выплат.
- Настройки каналов уведомлений для клиента, партнёра и Rooms, а также демо-очередь email/Telegram в админке.
- Информационные и юридические страницы с отдельным журналом пользовательских согласий.
- Центр миграции в админке: безопасный JSON-экспорт без паролей и сессий, проверка и контролируемый импорт.
- Счётчики по городам: предложение площадок для клиента, диапазон активной аудитории для партнёра и точная таблица в админке.

Первый серверный модуль уже работает локально на Fastify и TypeScript: города и их статистика, поиск помещений, карточка, отзывы, пересечение свободных окон, безопасная авторизация, профиль, заявки клиента и очередь партнёра. PostgreSQL 17 установлен, первоначальная схема применена, демо-каталог загружен, а локальный API читает данные через типизированные репозитории. При открытии сайта через локальный API frontend синхронизирует каталог, отзывы и заявки с PostgreSQL; стоимость и предоплата рассчитываются сервером. После локальной демо-оплаты удержание атомарно превращается в оплаченную бронь. GitHub Pages сохраняет встроенный каталог и работает независимо от компьютера.

### Запуск API

```powershell
cd server
npm install
npm run dev
```

После запуска сам сайт доступен на `http://127.0.0.1:3000/`, проверка сервера — на `/health`, публичные маршруты — на `/v1/*`. Проверки: `npm test`, production-сборка: `npm run build`.

## Важно

Каталог, опубликованные отзывы, авторизация, профиль, создание/чтение заявок клиента, очередь партнёра и серверный жизненный цикл предоплаты уже работают через API в локальном режиме. Подтверждение площадкой удерживает слот в PostgreSQL на 15 минут, конфликтующая заявка не может быть подтверждена, оплата закрепляет резерв без срока, а отклонение или истечение времени освобождает его. Демо-адаптер не принимает номер карты и автоматически отключается в production; перед реальными платежами его нужно заменить адаптером Сбер и подписанным webhook. Чаты, предложение другого времени, ручные записи партнёра и административные операции пока сохраняются в браузере. Для настоящего MVP также остаются восстановление пароля, онлайн-касса, серверная отправка email/Telegram, проверка юридических документов профильным юристом и реальные данные подключённых площадок.

Старые файлы `mobile.html`, `web.html`, `app/*` и standalone-макет пока оставлены как архив визуальных вариантов, чтобы не потерять дизайн-материалы.
